# Sidebar Submenu Implementation Plan

## Goal
Enhance the Sidebar component to support nested menu structures based on the "menus" array from the JWT token, allowing for multi-level navigation with expandable submenus.

## Current State Analysis
- Sidebar renders a flat list of menu items from `defaultData.modules`
- Supports collapsed and mobile views
- Uses static profile and dashboard items
- Navigation uses `navigate` and dispatches route changes

## Proposed Design
### Data Structure
- Extend `NavItem` interface to support nested children
- Transform the `menus` array into a tree structure for rendering

### UI/UX Design
- Top-level modules (e.g., "Pendaftaran") as expandable sections
- Use chevron icons to indicate expandable state
- Nested items indented with visual hierarchy
- Hover states and active highlighting for all levels
- Collapsed view: Show top-level icons only, expand on hover or click
- Mobile view: Full expanded tree with collapsible sections

### State Management
- Add state for tracking expanded/collapsed submenu states
- Persist expansion state per menu item
- Handle active route highlighting for nested items

### Implementation Steps
1. Update TypeScript interfaces to support nested menu structure
2. Create utility functions to transform JWT menus data into renderable tree
3. Add state management for submenu expansion
4. Modify rendering logic for expanded and collapsed views
5. Implement expand/collapse functionality with smooth animations
6. Update active route detection to work with nested routes
7. Handle permissions integration (if needed for menu filtering)
8. Test all view states (expanded, collapsed, mobile)

## Technical Considerations
- Maintain backward compatibility with existing flat menu structure
- Optimize performance for deep nesting (limit to 3-4 levels)
- Ensure accessibility (keyboard navigation, ARIA labels)
- Handle cases where menu data is missing or malformed

## Files to Modify
- `src/Components/Layout/Sidebar.tsx` - Main component changes
- Potentially add utility functions for menu transformation

## Testing Requirements
- Verify submenu expansion/collapse
- Test navigation to nested routes
- Confirm collapsed view behavior
- Mobile responsiveness
- Active state highlighting for nested items
